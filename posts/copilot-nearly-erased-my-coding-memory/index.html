Copilot Almost Erased My Coding Memory

5 min read • March 29, 2024

I have been writing code for almost 2 decades and in a few months Copilot almost erased my coding memory.

When I first read about Copilot, it was clear to me that it would make a huge impact on the development community. I was a little bit concerned reggarding using it on private projects, but this is something for another post.

I got access to a key I immediately set it up on my editor. The setup was impressive simple and fast, but do you know what else was also super fast? Yes, writing code down with it!

Breaking The Sound Barrier With The Keyboard

Writing code with Copilot enabled is literally like this: you write just a few characters and decide whether you want to accept or not and it’s suggesting. If you don’t want to accept the suggestion, you can just continue typing and it’ll suggest something else. When you accept—pressing TAB in my case—it writes the suggested code down.

Eu particularmente digito moderadamente rápido—120-140wpm—, eu também tenho atalho no teclado para quase tudo—raramente uso o mouse—, e escrevo codigo praticamente todos os dias desde 2009. Agora pegue tudo isso e some com o poder do Copilot, impossível de afirmar que ele não dá um boost significativo na hora de escrever codigo.

Depois de meses usando o Copilot eu notei um preucupante padrão: o quanto mais eu usava e aceitava suas sugestões, menos eu me lembrava como escrever certas coisas. Até que em um certo dia eu não me lembrava se quer como declarar uma variável em uma das linguagens que eu já trabalhava à anos.

Trabalhar com multiplas linguagens provavelmente faz parte do dia-a-dia de todos os desenvolvedores. Mesmo os projetos com apenas uma linguagem de programacao, sempre tem algo para dar suporte ao projeto como: arquivos de configuracao, scripts de automacao, gerenciador de dependencies, etc.

Como todos sabemos, quanto mais código a gente escrever em uma determinada linguagem, mais rápido nos ficamos em resolver problemas com ela. Não apenas no sentido de digitar mais rápido, mas também no sentido de que ficamos familiarizados com a syntax, funcoes nativas, depuracao de erros, etc.

Descobrindo o quanto o Copilot me afetou

Mesmo que voce precise ler a sugestão do Copilot para decidir se aceita ou não, além de não ter precisado usar o seu cerebro para pensar na solucao—porque o Copilot já fez isso para voce—, voce também não exercita sua memoria coginitiva porque tudo que voce precisa fazer é pressionar tab para que ele escreva para voce. Logo, quanto mais mais sugestões dele voce aceita, menos voce precisou pensar, menos voce precisou digitar, e mais voce esqueceu

Quando passamos muito tempo sem escrever em uma determinada linguagem, sempre acontece de esquecer alguns detalhes como funcões nativas e as vezes confundir a syntax—usar a syntax de outra linguagem—de coisas básicas como conditional statements, type declaration, entre outras. O problema é que eu estava escrevendo diariamente código na linguagem, ou, pelo menos, foi era isso que eu pensava.

Para ter certeza que não era apenas um dia daqueles dias que a gente mal consegue se concentrar e escrever codigo eu decidi que a primeira coisa que eu faria no dia seguinte seria desabilitar o Copilot e escrever algo simples, algo que eu já escrevi várias vezes e de várias formas: Um Lexer.

Criei um novo diretorio, e neste, um subdiretorio para cada linguagem que havia utilizado recentemente. Ao abrir o arquivo main de cada linguagem estava claro que eu de fato havia esquecido como escrever codigo. Foi um caos total, escrevendo syntax de C++ em Go, syntax de Javascript em PHP, entre outras.

The Importance of Active Coding

The role of continuous practice in skill retention Comparison of manual coding vs. AI-assisted coding

What I realized was that hands-on coding is crucial for maintaining and sharpening skills. It’s about more than just getting the code to work; it’s about the mental exercise involved in navigating the complexities of logic and structure. While Copilot can speed up the coding process, it shouldn’t replace the active engagement required to truly master the craft.

Finding a Path Forward

Strategies for balancing Copilot use with traditional coding Tips for maintaining and improving coding skills

Recognizing this, I decided to re-evaluate my approach. I now use Copilot more selectively, focusing on tasks where it can genuinely save time without undermining my learning. For other tasks, I make a conscious effort to code manually, ensuring that my skills remain intact. I’ve also set aside dedicated time for coding practice to reinforce my knowledge and abilities.

Conclusion

Reflecting on personal insights Encouragement for others facing similar challenges

This experience with Copilot taught me a valuable lesson: while AI tools are incredibly useful, they are no substitute for the foundational skills and mental agility that come with active practice. For anyone finding themselves in a similar situation, I encourage you to find a balance that allows you to harness the power of these tools while still nurturing your coding capabilities. Remember, it’s not just about writing code—it’s about understanding and owning every line.
